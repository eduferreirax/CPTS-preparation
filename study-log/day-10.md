# 🔄 Transition from CPTS to CWES

## 📌 Overview

This document explains my decision to transition my study focus from the **HTB CPTS (Certified Penetration Testing Specialist)** path to the **HTB CWES (Certified Web Exploitation Specialist)** path.

This was not a random change, but a strategic decision aligned with my long-term career goals.

---

## 🎯 Career Direction

My primary focus is:

- Application Security (AppSec)
- Web Security
- Bug Bounty Hunting

Instead of becoming a generalist in infrastructure and Active Directory environments, I am intentionally choosing to specialize in **web exploitation and application-layer vulnerabilities**.

---

## ⚖️ CPTS vs CWES (My Perspective)

### CPTS Focus:
- Network penetration testing
- Active Directory attacks
- Infrastructure exploitation
- Broad offensive security knowledge

### CWES Focus:
- Web application vulnerabilities
- API security
- Authentication & authorization flaws
- Real-world bug bounty scenarios

---

## 🧠 Reason for Transition

After reflecting on my interests and strengths, I realized:

- I enjoy **web hacking much more** than infrastructure/AD
- Modern attack surfaces are heavily focused on **web applications and APIs**
- Bug bounty programs are **primarily web-based**
- AppSec roles require deep understanding of **application logic and web vulnerabilities**

Because of that, CWES aligns much better with my goals.

---

## 🚀 Strategic Advantage

By focusing on CWES, I can:

- Develop **deep expertise in web exploitation**
- Build skills directly applicable to **bug bounty programs**
- Transition more naturally into **Application Security roles**
- Avoid unnecessary context-switching into areas I don't intend to specialize in

Instead of trying to cover all areas of offensive security, I am choosing to specialize in a domain that:
